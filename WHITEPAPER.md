## 释放机制（Proof of Contribution）

不挖矿，不烧电。通过生态贡献释放：

### 1. 存储贡献
- 节点存储链上数据 → 按存储量释放 SPARK
- 存得越多、越久 → 奖励越多

### 2. 带宽贡献
- 节点转发和传播交易 → 按带宽贡献释放 SPARK
- 保持在线、帮助网络通信 → 持续获得奖励

### 3. 验证贡献
- 节点参与交易验证和记账 → 释放 SPARK + 手续费分成
- 维护网络安全和一致性

### 释放曲线
- 总量固定：1万亿 SPARK
- 生态释放池 6000亿，分 10 年释放
- 每 2 年减半一次（Year1-2: 50%, Year3-4: 25%, Year5-6: 12.5%...）
- 防通胀，保证长期价值

### 早期空投
- 项目上线 GitHub 仓库
- 前 10,000 名安装用户每人空投 100,000 SPARK
- 安装即送，零门槛，拉动早期用户增长

## 节点分级

### 全节点（基础设施提供者）
- 提供服务器，存储全链数据，验证所有交易
- 奖励最多
- 门槛：稳定服务器 + 足够存储

### 轻节点（普通 AI / OpenClaw 用户）
- 只存自己相关数据
- 帮忙转发交易、贡献带宽
- 中等奖励
- 门槛：跑 OpenClaw 即可

### 微节点（手机/轻设备）
- 挂着在线，贡献少量带宽
- 少量奖励
- 门槛：装个 app

### 防垄断机制
- 单节点奖励有上限
- 节点数量 > 单节点算力（鼓励去中心化）
- 早期参与者加成

## 产品形态
- OpenClaw 插件，不是独立 app
- 用户安装 OpenClaw → 装 Spark 插件 → 自动变成节点
- 无账号系统，纯去中心化
- 安装时自动生成钱包 + 助记词（12/24词）
- 用户自行备份助记词，换设备用助记词恢复
- 助记词丢失 = 钱包丢失（不可找回）
- 设备指纹仅用于防作弊（同一台机器不能跑多个节点）
- 支持多钱包管理：主钱包（节点奖励）+ 用户自建子钱包（分类管理）
- 节点奖励只进主钱包，防止刷币

## 防作弊机制

### 1. 伪造存储
- 链随机抽查，要求节点限时返回指定数据的证明
- 答不上来扣币罚款

### 2. 伪造带宽
- 只计算有效交易的转发
- 同地址自发自收不算
- 同 IP/同网段节点互相转发打折或不计

### 3. 刷号薅空投
- 一个 OpenClaw gateway ID 绑定一个钱包
- 设备指纹锁定，同一台机器只能一个节点

### 4. 女巫攻击（大量假节点）
- 成为节点需质押 SPARK，作弊没收
- 节点间互相验证，异常行为自动举报

### 5. 验证串通
- 验证者随机分配
- 同一笔交易多个不相关节点交叉验证

### 核心原则
- 作弊成本 > 收益
- 渐进信任：新节点奖励低，运行越久越稳定奖励越高
- 长期老实干活的赚得多，短期刷量的赚不到
